# Knowledge Base Metadata

## Architecture Domain

### 1. Business Architecture
**Path:** `export/architecture/business`
**Name:** Enterprise Business Architecture & Domain-Driven Design
**Description:** This knowledge base focuses on strategic business architecture, Domain-Driven Design (DDD), and aligning technical solutions with business capabilities. It contains distilled rules on bounded contexts, ubiquitous language, strategic modeling, and translating business requirements into technical domains. Use this when designing high-level enterprise systems, determining microservice boundaries based on business functions, and modeling complex business logic.

### 2. Application Architecture
**Path:** `export/architecture/application`
**Name:** Application Architecture & Software Design Patterns
**Description:** A comprehensive repository of software application architecture patterns, principles, and best practices. It covers structural methodologies like Clean Architecture, Hexagonal Architecture, Onion Architecture, and SOLID principles. It also details the architectural characteristics and tradeoffs between Monolithic systems and Microservices. Query this when deciding on project folder structures, boundary layers, dependency injection, and overall software design before writing implementation code.

### 3. Data Architecture
**Path:** `export/architecture/data`
**Name:** Data Architecture & Modeling Principles
**Description:** Dedicated to rules and best practices for data modeling, database schema design, and data flow patterns. This includes guidance on relational (SQL) and non-relational (NoSQL) database design, caching layers, event sourcing, CQRS (Command Query Responsibility Segregation), and distributed data management. Use this when designing schemas, planning data access layers, or managing transactions and migrations safely.

### 4. Integration Architecture
**Path:** `export/architecture/integration`
**Name:** Systems Integration & API Architecture
**Description:** Focuses on the architectural best practices for system-to-system communication. It contains distilled knowledge on designing robust APIs (REST, GraphQL, gRPC), setting up message brokers (Kafka, RabbitMQ) and event-driven architectures, utilizing webhooks, and making tradeoffs between synchronous and asynchronous communication patterns. Consult this when connecting disparate systems, designing public or internal APIs, or building event streaming platforms.

### 5. Technology Architecture
**Path:** `export/architecture/technology`
**Name:** Cloud Infrastructure & Deployment Architecture
**Description:** A repository of infrastructure and cloud-native architecture principles. It covers deployment topologies, High Availability (HA) design, Disaster Recovery (DR) strategies, scaling models (horizontal vs. vertical), and containerization/orchestration best practices (Docker, Kubernetes). Query this when designing cloud environments, planning infrastructure resilience, or structuring scalable deployment architectures.

## Framework Domain

### 6. React Framework
**Path:** `export/framework/react`
**Name:** React Component Architecture & Best Practices
**Description:** Contains distilled best practices, anti-patterns, and architectural rules for React.js development derived from domain-expert literature. It provides authoritative guidance on component lifecycles, state management logic, solving prop drilling, custom hooks, and UI rendering optimizations. Use this when writing or reviewing React code to ensure it follows idiomatic, clean code principles.

### 7. Next.js Framework
**Path:** `export/framework/nextjs`
**Name:** Next.js Application Architecture & Rendering
**Description:** Retrieves official and expert-derived best practices for Next.js applications. It covers architectural decisions regarding routing (App router vs. Pages router), rendering strategies (SSR, SSG, ISR, CSR), data fetching methods, caching, middleware, and Next.js-specific performance optimizations. Query this when structuring Next.js applications or choosing the correct rendering methodology for specific pages.

### 8. Node.js Runtime
**Path:** `export/framework/node`
**Name:** Node.js Runtime & Server-Side Patterns
**Description:** Fetches foundational rules and performance best practices for the Node.js runtime. This knowledge base covers server-side JavaScript architecture, mastering the event loop, managing streams and buffers, efficient file system handling, and avoiding blocking asynchronous I/O operations. Use this when writing high-performance Node.js backend services and ensuring memory-efficient code execution.
